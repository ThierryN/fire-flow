Phoenix Rebuild takes a messy "vibe coded" project and rebuilds it clean from extracted intent. The source folder is never modified — the rebuild goes to a new target folder.

Intent extraction is the key innovation: instead of literally translating messy code, the fire-phoenix-analyst agent reverse-engineers what the developer was trying to do. Every feature is classified as INTENDED, ACCIDENTAL, WORKAROUND, or ABANDONED. The rebuild preserves intent while removing accidental complexity.

Phoenix Score (PX-1 to PX-5): Feature Parity (30%) + Edge Case Coverage (25%) + Dependency Compatibility (20%) + Accidental Complexity Removal (15%) + Architecture Improvement (10%). Score of 90%+ = APPROVED.

Why files instead of memory? Claude has no memory between sessions — each conversation starts blank. Dominion Flow externalizes all state into markdown files in .planning/. The agent is stateless by design; the files carry the memory. This is the same pattern databases use: the process is stateless, the storage persists.

Breath execution solves dependency ordering: tasks within a breath run in parallel (multiple subagents at once), but the next breath only starts when all tasks in the current breath are complete. Maximum parallelism, zero dependency violations.

Circuit breaker intelligence (v12.0) classifies stuck states into 6 types (stall, spin, degrade, blocked, thrash, drift) with Google X-style kill conditions. When the agent is genuinely stuck, it stops cleanly instead of burning tokens.

Phoenix Rebuild (v12.2) is the first AI coding tool that can take a messy "vibe coded" project and rebuild it clean from extracted developer intent. The 6-phase pipeline (AUTOPSY, INTENT, CLARIFY, VISION, REBUILD, COMPARISON) preserves what matters and removes accidental complexity.
